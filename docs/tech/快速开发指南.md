# 快速开发指南（Vue + Tailwind CSS + shadcn-vue + lucide 图标）

> 面向本仓库前端子工程 `web/` 的最小上手手册。

## 1. 开发环境
- Node.js ≥ 18（推荐 LTS）
- 包管理：npm（如网络较慢，可切换 `npmmirror` 源）

切换镜像源（可选）：
```bash
npm config set registry https://registry.npmmirror.com
npm config get registry
```

## 2. 初始化与运行
```bash
# 进入前端工程
cd /media/vdc/github/hubble-pad/web

# 安装依赖（首次）
npm install

# 开发模式
npm run dev   # 默认 http://localhost:5173

# 构建产物
npm run build # 产出到 web/dist

# 本地预览构建产物
npm run preview
```

## 3. 技术栈与目录
- 栈：Vue 3 + TypeScript + Tailwind CSS v4 + shadcn-vue（Radix 生态）
- 图标：unplugin-icons，图标源使用 `lucide`（与 shadcn 生态一致）
- 关键文件：
  - `web/vite.config.ts`：注册 `unplugin-icons` 与自动导入（前缀 `icon-lucide-*`）
  - `web/src/style.css`：Tailwind v4 入口（`@import "tailwindcss";`）
  - `web/src/App.vue`：应用入口页面（可直接开始开发）

目录示例：
```
web/
  src/
    App.vue
    style.css
  vite.config.ts
  package.json
```

## 4. 图标最小示例（lucide）
- 已在 `vite.config.ts` 配置自动按需导入，直接在模板里使用：
```vue
<template>
  <div class="flex items-center gap-2 text-gray-700">
    <icon-lucide-bolt class="w-5 h-5" />
    <span>Hubble Pad</span>
  </div>
</template>
```
- 其他图标名参考 lucide 集合，类名前缀统一为 `icon-lucide-`。

<template>

## 6. 常见问题
- 端口占用：修改 `npm run dev` 输出的调试地址或在 `vite` 配置 `server.port`。
- 安装缓慢：切换 npm 镜像源（见上文），或使用公司内源。
- 图标不显示：确认已使用 `icon-lucide-*` 命名，且 `vite.config.ts` 中已启用 `unplugin-icons` 与 `Components` 自动导入。
- 样式无效：确认 `src/style.css` 仅包含 `@import "tailwindcss";` 并已在入口（`main.ts` 默认导入 `style.css`）。
