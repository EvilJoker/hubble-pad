# 文档总览

本目录用于存放所有文档，写作建议：轻、短、准

## 特性文档写作建议
- idea.md：初期需求与理念梳理（可选保留，用于历史追踪）
- 方向层（稳定, 设计总纲与统一抽象，不轻易变更）
  - 存放：`docs/design.md`
  - 内容：愿景/边界、统一抽象（WorkItem 最小字段）、UI 信息架构、数据策略
- 方案层（可变, RFC 提案 子主题的设计）
  - 存放：`docs/rfcs/`（一主题一 RFC，草案→稳定）
  - 内容：接口/映射、权衡取舍、失败与回滚、影响面；结论回链 design.md
- 实现层（易变, 贴近代码细粒度特性的文档）
  - 存放：`docs/tech/框架设计.md`(子特详设文档)
  - 内容：启动/脚本/环境变量、目录规范、调试与预览
- Roadmap（按版本）
  - 存放：`docs/version/roadmap-vX.Y.Z.md`
  - 内容：版本开发计划与跟踪文档
  - 策略：版本完成后冻结保留，不再更新；下一版本新建对应 roadmap

## Roadmap 管理策略
- 命名规则：`roadmap-v1.0.0.md`、`roadmap-v2.0.0.md` 等，按版本号命名
- 分支管理：每个版本功能在新分支（如 `release/v1.0.0`）中开发
- 文档保留：完成后不删除，作为历史记录保留在 docs/ 目录，不再更新
- 下一版本：新版本启动时创建新的 roadmap 文件，旧版本 roadmap 冻结归档

说明：总纲只放稳定内容；具体方案与实现细节以链接方式引用到 RFC 与代码文档，避免频繁变更影响总纲稳定性。
